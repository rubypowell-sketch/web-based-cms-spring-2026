### Austin Community College | Department of Technical Communications

**Welcome, Content Strategists!**

This repository is your "Classroom in the Cloud." It contains the source code, content, and automation tools you will use to build your portfolio this semester.

---

## 🚀 Getting Started (Day 1)

**Step 1: Fork this Repository**
* Click the **Fork** button (top right corner of this page).
* This creates a copy in your own GitHub account. You will do all your work in *your* copy.

**Step 2: Launch Your Environment**
* Click the green **Code** button.
* Select the **Codespaces** tab.
* Click **Create codespace on main**.
* *Note:* The first launch may take 2-4 minutes to build. Grab a coffee! ☕

**Step 3: Verify Your Tools**
* Once the environment loads, check the **Extensions** tab on the left.
* You should see tools like **Markdown All in One** and **Code Spell Checker** installed automatically.

## 📂 Repository Map
*Where is everything?*

| Folder | Description |
| :--- | :--- |
| **`docs/`** | **The Live Site.** This is where your actual homework lives. Anything you put here will be published to the web. |
| **`legacy-content/`** | **Project 1 Source Material.** This folder contains "messy" files (Word docs, PDFs) that you must audit and migrate. |
| **`specs/`** | **Project 2 Requirements.** This folder contains the engineering tickets and product specs for the new features you will document. |
| **`.devcontainer/`** | **System Config.** This magic folder tells GitHub how to build your computer. *Do not touch this unless instructed.* |
| **`.github/`** | **Automation.** Contains the workflows that check your spelling and publish your site. |
